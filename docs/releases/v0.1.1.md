# v0.1.1

Incremental hardening release focused on reproducibility, safety, and docs consistency.

## Highlights

- **Per-project clone workflow added**
  - `scripts/new_project_bootstrap.sh` now supports:
    - `--mode clone`
    - `--repo-url`
    - `--ref` (pin baseline by tag/branch)

- **Bootstrap metadata tracking**
  - New projects now receive `.bootstrap/BASELINE_META.json` for source/ref traceability.

- **OpenClaw optional setup made non-destructive**
  - `scripts/setup_openclaw_optional.sh` now preserves existing memory files (uses `touch`, avoids truncation).

- **Repository validation pipeline added**
  - `scripts/validate_repo.sh`
  - GitHub Actions workflow: `.github/workflows/validate.yml`
  - Checks shell/python syntax, privacy patterns, and unresolved placeholders.

- **Multilingual README synchronization**
  - EN/KO/JP/ZH structure aligned
  - Added “recommended strategy: per-project clone” section across all languages.

## Upgrade Notes

- Existing users can continue with local mode:
  ```bash
  bash scripts/new_project_bootstrap.sh /path/to/new-project
  ```

- For reproducibility, switch to clone mode:
  ```bash
  bash scripts/new_project_bootstrap.sh /path/to/new-project --mode clone --ref v0.1.1
  ```
